01 요약과 배경
맥락

문제 정의

NDX PRO 신규 사용자는 NGSI-LD 기반의 Data Model 생성 단계에서 높은 이탈률을 보입니다. @context 설정, 속성 타입 정의 등 플랫폼 고유 개념에 대한 사전 지식 없이는 첫 Model 생성이 불가능한 구조입니다.

왜 지금인가

Hyundai 도메인 확장에 따른 신규 사용자 온보딩 건수 증가 예정. 현재 온보딩 가이드 부재로 지원 요청이 집중되고 있으며, Data Model Studio 출시 전 온보딩 경험 정비가 필수입니다.

목표 지표

지표현재목표
첫 Model 생성 완료율미측정70% 이상
온보딩 완료 시간미측정20분 이내
지원 요청 건수기준치 설정 필요30% 감소
OnPlan AI 기획자 의견
PM과 개발자의 문제 인식이 다릅니다. PM은 UX 복잡도(온보딩 경험)를 강조한 반면, 개발자는 Translator 연결 구조의 분리가 근본 원인이라 봤습니다. 둘 다 유효하나 온보딩 개선은 UX 레이어에서 먼저 해결하고, 구조 개선은 Data Model Studio에서 병행하는 것을 권장합니다.

03 핵심 사용자 여정
정의

As-Is

NDX PRO 첫 접속 → 빈 화면 → Data Model 메뉴 탐색 → Schema 탭 발견 → @context 오류 반복 → 지원 요청 또는 이탈

To-Be

NDX PRO 첫 접속 → 온보딩 가이드 투어 시작 → Data Model 생성 (인라인 힌트 포함) → Ingest 연결 유도 → 첫 Entity 확인 → 온보딩 완료

사용자가 얻는 가치

NGSI-LD 스펙 학습 없이 첫 Model 완성. 실수 시 안전한 롤백. Data Model → Ingest → Entity 플로우 직관적 이해.

OnPlan AI 기획자 의견
현재 플로우에서 가장 이탈이 많은 구간은 @context 설정입니다. 이 구간에서 인라인 설명과 기본값 자동 채움만 제공해도 완료율이 크게 오릅니다. 전체 투어보다 이 단 하나의 개선이 우선순위 1순위입니다.

OnPlan AI 기획자 의견
P0 3개 기준 2개월 내 가능합니다. 도메인 템플릿은 Hyundai 케이스가 있으므로 P1이지만 실질적으로 P0에 가깝습니다. AI 스키마 자동 생성은 NDX PRO의 핵심 차별화 기능이 될 수 있으나, MVP 범위를 벗어납니다.
의견 충돌 · 결정 필요
도메인 템플릿을 P0으로 올릴 것인가
PM은 Hyundai 도메인 대응을 위해 템플릿이 P0이어야 한다는 입장. 개발자는 인라인 힌트 구현만으로도 일정이 빡빡하다는 의견.
OnPlan AI 현재 권고
템플릿 1종(Hyundai 기본 Vehicle Model)만 P0에 포함하고 나머지 2종은 P1으로 분류하면 일정 영향 없이 절충 가능합니다.
AI 수정 제안
템플릿 1종을 P0에 포함하면 인라인 힌트 구현과 병렬 진행 가능합니다.

정리하면:
— 안 A: 템플릿 없음 · P0 3개 = 2개월
— 안 B: 템플릿 1종 포함 · P0 4개 = 2개월 (병렬)
— 안 C: 템플릿 3종 포함 · P0 5개 = 3개월

안 B를 권장합니다. 어떻게 하시겠습니까?

OnPlan AI 기획자 의견

06 사용자 플로우
설계

플로우 1: 첫 온보딩 (신규 사용자)

첫 로그인 → 온보딩 투어 시작 (Step 1: Data Model 소개) → Step 2: Schema 생성 실습 → Step 3: Ingest 연결 안내 → 완료 → 대시보드

플로우 2: Data Model 생성

Data Model 메뉴 → [+ 새 모델] → 이름/설명 입력 → Schema 편집 (인라인 힌트 활성) → 저장 → Ingest 연결 유도 배너 노출

플로우 3: 관리자 온보딩 현황 확인

관리자 로그인 → 온보딩 현황 메뉴 → 사용자별 단계 진행률 → 미완료 사용자 리마인드 발송

07 배포 계획
실행

Phase 1 — MVP (1개월)

@context 인라인 힌트, Ingest 연결 유도 배너. Hyundai 파일럿 적용.

Phase 2 — 온보딩 투어 (2개월)

3단계 가이드 투어, 도메인 템플릿 1종. 완료율 측정 시작.

Phase 3 — 관리자 대시보드 (3개월)

온보딩 완료율 대시보드, 템플릿 라이브러리 3종. 정식 배포.

OnPlan AI 기획자 의견
Phase 1이 가장 중요합니다. 인라인 힌트 하나로 완료율이 얼마나 오르는지 먼저 측정하고, 그 데이터로 Phase 2~3 우선순위를 재조정하는 것을 권장합니다. 투어 없이도 힌트만으로 충분할 수 있습니다.

기술 제약

기존 Schema 편집기 컴포넌트에 오버레이 추가 방식. MongoDB Entity 스키마와 충돌 없이 구현 필요. 힌트 콘텐츠는 JSON 파일로 외부화하여 업데이트 가능하게 관리.

NGSI-LD 스펙을 몰라도 바로 시작할 수 있는 NDX PRO 온보딩 가이드.

해결하는 문제

NDX PRO는 강력하지만 첫 진입 장벽이 높습니다. @context, 속성 타입 등 NGSI-LD 고유 개념을 모르면 첫 Data Model조차 만들기 어렵습니다. 이 개선으로 누구나 20분 안에 첫 Model을 완성할 수 있습니다.

핵심 기능 3가지

1. 3단계 가이드 투어 — 첫 접속 즉시 시작, 스킵 가능
2. 인라인 힌트 — Schema 편집 시 NGSI-LD 속성 타입 자동 안내
3. Ingest 연결 유도 — Model 완성 후 다음 단계 자동 안내

차별점

단순 매뉴얼이나 영상이 아닌, 실제 작업 화면 안에서 맥락에 맞게 제공되는 인라인 가이드. 학습 없이 실무에 바로 적용 가능.
