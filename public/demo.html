경영 판단용

요약

발주 누락으로 인한 월 500만원 손실을 제거하기 위한 경량 발주 관리 시스템입니다. 기존 ERP(3,000만원+) 대비 500만원 이내, 4개월 내 구축 가능합니다.

투자 대비 효과

항목현재도입 후
월 발주 누락3~4건 (약 500만원 손실)0건
월말 정산 소요3일0.5일 (80% 절감)
인수인계 기간2~3주시스템에 기록 잔존

리스크

최대 리스크는 거래처의 시스템 사용 거부입니다. 이메일/SMS 폴백을 P0에 포함하여 대응합니다. 거래처가 시스템을 쓰지 않아도 내부 발주 관리는 정상 작동합니다.

의사결정 필요 사항

1. 거래처를 시스템 사용자로 포함할지 여부 (포함 시 +300만원, +2개월)
2. 일정 3개월 고수 시 대시보드 필터 간소화 수용 가능 여부

Summary

A lightweight order management system to eliminate ~$5K/month in losses from missed purchase orders. Buildable within $50K and 4 months, compared to traditional ERP ($300K+).

ROI

MetricCurrentAfter Implementation
Missed Orders/Month3-4 (~$5K loss)0
Month-End Settlement3 days0.5 days (80% reduction)
Handoff Period2-3 weeksRecords persist in system

Risks

Top risk is vendor refusal to adopt the system. Email/SMS fallback is included in P0. Internal order management works regardless of vendor adoption.

Decisions Needed

1. Include vendors as system users? (If yes: +$30K, +2 months)
2. Accept simplified dashboard filters if 3-month timeline is mandatory?

기술 제약

거래처 관리와 발주 관리는 DB 레벨에서 분리. 추후 거래처 포털 확장 대비.
알림 시스템은 외부 API 의존 (SMS: NHN Cloud 또는 유사). 실패 핸들링 필수.
동시 편집 충돌: 낙관적 잠금 적용.

일정 기획자 의견

P0 3개 기준 4개월 권장. 3개월 강행 시 대시보드 필터/검색 간소화 필요.

P0 Feature Spec

FeatureDetailAcceptance Criteria
Order CreationSelect vendor → Enter items (multiple) → Price, qty, delivery date → Preview → SendComplete send in ≤2 clicks
Vendor Confirmation AlertEmail + SMS on send. Auto-remind once after 24h if unconfirmed.Notify owner on alert delivery failure
Status DashboardStatus-based list (Pending/Confirmed/Delivered/Complete). Basic list, filters in P1.Load last 50 orders in <1 second

Technical Constraints

Vendors and Orders separated at DB level for future vendor portal extensibility.
Notification system depends on external API (SMS: Twilio or similar). Failure handling required.
Concurrent edit conflicts: optimistic locking.

OnPlan Timeline Opinion

4 months recommended for 3 P0 features. If forced to 3 months, dashboard filters must be simplified.

DESIGNUX 요구사항
디자인 착수용

핵심 사용자

1차 사용자: 구매 담당자 (일 5~10건 발주 생성, 가장 빈번한 사용자)
2차 사용자: 운영 팀장 (현황 확인, 주 1~2회 리포트 조회)

UX 원칙

1. 교육 없이 즉시 사용 가능해야 합니다 (대표 요구사항).
2. 신규 발주 생성이 가장 빈번한 액션이므로 대시보드에서 2클릭 이내 도달.
3. 현장 직원 대상이므로 정보 밀도보다 명확한 상태 표시 우선.

핵심 플로우 (우선 설계 대상)

1. 신규 발주 생성: 대시보드 → [+새 발주] → 거래처 → 품목 → 확인 → 발송
2. 발주 현황 확인: 대시보드 → 상태별 목록 → 상세
3. 거래처 확인 응답: (거래처 측) 알림 링크 → 발주 내용 → 확인 버튼

사용자 여정

As-Is: 엑셀 → 카톡 → 구두 확인 → 수기 기록
To-Be: 시스템 생성 → 자동 알림 → 시스템 확인 → 자동 상태 업데이트

참고

모바일 대응 필요 (현장 사용). 좌측 사이드바(데스크톱) / 하단 탭(모바일) 네비게이션.

Key Users

Primary: Procurement officer (5-10 orders/day, most frequent user)
Secondary: Operations manager (status checks, 1-2 report views/week)

UX Principles

1. Usable immediately without training (CEO requirement).
2. New order creation is the most frequent action — reachable within 2 clicks from dashboard.
3. For field staff: prioritize clear status indicators over information density.

Key Flows (Design Priority)

1. New Order: Dashboard → [+New Order] → Vendor → Items → Confirm → Send
2. Order Status: Dashboard → Status list → Detail
3. Vendor Confirmation: (Vendor side) Alert link → Order details → Confirm button

User Journey

As-Is: Excel → Messenger → Verbal confirmation → Manual record
To-Be: System creation → Auto alert → System confirmation → Auto status update

Notes

Mobile support required (field usage). Left sidebar (desktop) / bottom tabs (mobile) navigation.

SALES제품 소개 요약
고객 커뮤니케이션용

한 줄 소개

엑셀과 카톡으로 발주를 관리하는 중소 제조업체를 위한 경량 발주 관리 시스템.

해결하는 문제

거래처 10곳 이상이면 엑셀로 감당 안 됩니다. 발주 누락, 단가 착오, 납기 지연이 반복되고 담당자 바뀌면 인수인계가 끊깁니다. 이 시스템은 발주 생성부터 거래처 확인, 현황 추적까지 한곳에서 처리합니다.

핵심 기능 3가지

1. 발주서를 1클릭으로 생성·발송하고, 거래처에 자동 알림
2. 거래처가 확인하지 않으면 24시간 후 자동 리마인드
3. 전체 발주 현황을 대시보드에서 한눈에 확인

경쟁 대비 차별점

ERP는 너무 비싸고(3,000만원+) 복잡합니다. 이 시스템은 발주 프로세스에만 집중하여 500만원 이내, 교육 없이 바로 사용 가능합니다.

고객 대상

거래처 10~50곳 규모의 중소 제조업체. 구매팀 1~5명. 현재 엑셀+메신저로 발주 관리 중인 곳.

One-Liner

A lightweight order management system for SMB manufacturers still using spreadsheets and messaging apps.

Problem Solved

Beyond 10 vendors, spreadsheets can't keep up. Missed orders, pricing errors, and delivery delays pile up. Staff turnover breaks handoffs. This system handles order creation, vendor confirmation, and status tracking in one place.

3 Key Features

1. Create and send purchase orders in 1 click with automatic vendor alerts
2. Auto-remind vendors after 24 hours if unconfirmed
3. Full order status dashboard at a glance

Competitive Edge

ERP is too expensive ($300K+) and complex. This system focuses solely on the ordering process — under $50K, usable immediately without training.

Target Customer

SMB manufacturers with 10-50 vendors. 1-5 person procurement teams. Currently managing orders via spreadsheets + messaging apps.
